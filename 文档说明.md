## 🔑 核心概念澄清

### 架构理解（重要！）

```
┌─────────────────────────────────────────────┐
│          中国用户（浏览器）                  │
│          无需代理，正常访问                  │
└────────────────┬────────────────────────────┘
                 │ HTTP
                 ↓
┌─────────────────────────────────────────────┐
│          前端网页（React）                   │
│          可部署在中国（阿里云/腾讯云/CDN）   │
│          用户访问速度快                      │
└────────────────┬────────────────────────────┘
                 │ HTTP API
                 ↓
┌─────────────────────────────────────────────┐
│          后端服务器（Node.js）               │
│          需要访问 Google API                 │
│          方案1: 海外服务器（推荐）           │
│          方案2: 国内服务器 + 代理            │
└────────────────┬────────────────────────────┘
                 │ HTTPS (需要能访问 Google)
                 ↓
┌─────────────────────────────────────────────┐
│          Google Vertex AI                   │
│          在中国被墙                          │
└─────────────────────────────────────────────┘
```

### 关键点

1. **前端可以在中国**
   - 普通网页，用户浏览器直接访问
   - 可以部署在阿里云 OSS、腾讯云 COS、国内 CDN
   - 无需代理

2. **后端需要访问 Google API**
   - 可以部署在海外服务器（推荐，无需代理）
   - 或者国内服务器 + HTTP 代理
   - 只有后端需要代理，用户不需要

3. **用户体验**
   - 中国用户访问前端网页：快速（国内服务器）
   - 前端调用后端 API：正常速度
   - 后端调用 Google AI：需要能访问 Google
